## Interpretation

- Restoring JSX parity mattered.
- Opaque hero references outperformed the earlier alpha-cutout workflow.
- The white-background hero source was the best starting point for this full-body pass because it preserved Arthur's face and beard more reliably than the green-background variant.
- `Seed 1337` was the strongest TRELLIS seed in the white-background sweep.
- Lowering TRELLIS export decimation from `200000` to `80000` preserved the look while materially reducing triangle count, so `A14` is the best raw full-body result so far.
- Blender MCP decimation was validated successfully. The `40k` variant is the safest low-poly follow-up, and even `20k` remained close enough to prove the postprocess path works.
- A separate head pass is not necessary at this stage because the full-body result is now strong enough to continue the low-poly workflow directly.
- Weapons need their own preprocessing rule. The original glowing sword proxy was not clean enough; a flat-background, tight-cropped weapon input produced a much better TRELLIS result.
- The equipment pipeline now works end to end: `reference -> TRELLIS sword -> Blender attach -> combined GLB export -> QA renders`.
- Equipment placement cannot be approved from a single screenshot. Front, side, oblique, and user-like checks are required because a pose that looks correct in one angle can still read as detached or floating in another.
- `A22` showed that internal multi-angle checks still are not enough if the grip does not convincingly overlap the hand in the exact way the user expects. Final equipment approval still requires explicit user confirmation.
- The next missing quality step is pose-aware attachment. Arthur's current idle hand geometry is too open to sell a perfect grip, so future sword passes should add either a hand-pose adjustment, a socket-based placement standard, or a lightweight rig/pose step before final export.
- The easy enemy roster now has a clean dungeon replacement set that still fits the existing four gameplay family slots: `SkeletonWarrior`, `GoblinCharger`, `SlimeSpitter`, and `CaveBat`.
- For these enemy-sized assets, one strong seed on a clean opaque-green reference was enough. The first `1337` pass produced usable raws for all four families without additional seed sweeps.
- The Blender low-poly cleanup path is stable for this batch. Accepted targets are `24k` for the two bipeds, `10k` for the slime, and `14k` for the bat, with verified re-import renders on each exported GLB.
