# T66 Full Audit ‚Äî 2026-05-04

Read-only audit of `c:\UE\T66` (Unreal Engine 5.7, multi-module: T66, T66Mini, T66TD, T66Idle, T66Deck, T66Editor) covering three concerns:

1. Things that hurt performance on low-end PCs
2. Things that should be data-driven but currently aren't
3. Things that could cause crashes

Severity scale: **CRITICAL / HIGH / MEDIUM / LOW**.

---

## 0. Project context

- Engine: UE 5.7. Project file: `T66.uproject`.
- Modules: 6 (5 runtime + 1 editor). ~684 C++ source files.
- Networking: Steam (`OnlineSubsystemSteam`) + a Vercel HTTP backend (`T66BackendSubsystem`).
- Plugins enabled: `ProceduralMeshComponent`, `OnlineSubsystemSteam`, `SocketSubsystemSteamIP`.
- Configs present: `DefaultEngine.ini`, `DefaultGame.ini`, `DefaultInput.ini`, `DefaultEditor.ini`, plus four custom `.ini`s for Player Experience / Stage Progression / Trap Tuning / RNG.

- Existing data-driven layer (good ‚Äî mirror these):
  - `FT66PlayerExperienceDifficultyTuning` rows in `DT_PlayerExperience` (loaded by `T66PlayerExperienceSubSystem.cpp:7, 52`)

## 1. Performance ‚Äî things that will hurt low-end PCs

### 1a. Per-frame Tick work that scales with actor count

- **HIGH** ‚Äî `Source\T66\Gameplay\T66EnemyBase.cpp:892-1143` ‚Äî every live enemy ticks and calls `T66ResolveClosestPlayerPawn` (936-940), which iterates all PlayerControllers (172-199). Cost is `O(enemies √ó players)` every frame. Cache nearest-player on the GameMode/subsystem and refresh on a 5‚Äì10 Hz timer.
- **HIGH** ‚Äî `Source\T66\UI\HUD\T66GameplayHUDWidget.cpp:51-195` ‚Äî `NativeTick` runs `RefreshSpeedRunTimers`, throttled DPS, `PresentationController.Tick`, and during attack-lock writes crosshair transform every frame (107-124) plus `FString::Printf ‚Üí SetText` (141-162). Drive crosshair from PC events; only call `SetText` when the displayed value actually changes.
- **MEDIUM** ‚Äî `Source\T66Mini\Private\Gameplay\T66MiniEnemyBase.cpp:218-329` ‚Äî same "find nearest pawn" per Mini enemy per frame (260, 411-417). Same fix.
- **MEDIUM** ‚Äî `Source\T66\Gameplay\T66HeroBase.cpp:580-758` ‚Äî hero `Tick` does ground line traces (611-661), animation-state evaluation with `PlayAnimation` (699-731), and cooldown bar UMG updates (737-758).
- **MEDIUM** ‚Äî `Source\T66\Gameplay\T66HeroProjectile.cpp:68-102` ‚Äî homing projectiles tick + rewrite velocity each frame. Use `ProjectileMovement` homing where possible and pool projectiles.
- **MEDIUM** ‚Äî `Source\T66\Gameplay\T66GameMode.cpp:1125-1201` ‚Äî non-tower path runs `MaintainPlayerTerrainSafety()` every frame (1137-1140) while tower uses a 0.1s accumulator. Throttle non-tower the same way.
- **MEDIUM** ‚Äî `Source\T66\Gameplay\T66EnemyBase.cpp:897` ‚Äî every enemy constructs an `FLagScopedScope` (`T66LagTrackerSubsystem.h:92-127`). Gate construction behind `IsEnabled()` first.
- **MEDIUM** ‚Äî `Source\T66\Gameplay\Traps\T66FloorSpikePatchTrap.cpp:299-350` ‚Äî during warning/rise, updates all spike instance transforms each tick (341-349, plus `MarkRenderStateDirty` 459-491). Warning phase also spawns ~12 Niagara pixels every ~0.08s (312-337).
- **MEDIUM** ‚Äî `Source\T66\UI\T66VendorOverlayWidget.cpp:1942, 1960-1979` ‚Äî 30 Hz timer calling `SetWidthOverride` triggers a Slate layout pass each tick. Drop to 10‚Äì15 Hz.
- **MEDIUM** ‚Äî `Source\T66\UI\Components\T66LeaderboardFilterButton.cpp:77-145` ‚Äî `NativeTick` drives a sprite's MID scalar every frame. Use a brush animation or sprite-FPS timer.
- **MEDIUM** ‚Äî `Source\T66\UI\Gambler\T66GamblerOverlayWidget_Cheat.cpp:271, 379` ‚Äî ~30 Hz coin spin timer; same critique.
- **MEDIUM** ‚Äî `Source\T66Idle\Private\UI\Screens\T66IdleMainMenuScreen.cpp:165-172, 788-817` ‚Äî idle simulation runs every frame from a UMG screen tick.
- **LOW** ‚Äî `Source\T66\UI\T66QuickArcadeWidget.cpp:309`, `T66GoldMinerArcadeWidget.cpp:350`, `T66WhackAMoleArcadeWidget.cpp:351`, `T66TopwarArcadeWidget.cpp:371` ‚Äî 0.04‚Äì0.05s repeating timers in arcade minigames.
- **LOW** ‚Äî `Source\T66\Gameplay\T66EnemyBase.cpp:938-939` ‚Äî `UE_LOG(*GetName())` in tick path.

## 3. Crash risks

### 3.1. HTTP / async lifetime

- **HIGH** ‚Äî `Source\T66\Core\T66WebImageCache.cpp:58` ‚Äî `Request->OnProcessRequestComplete().BindLambda([this, CapturedUrl](...){ OnDownloadComplete(...); })`. Raw `this` capture on HTTP completion. Use-after-free if subsystem torn down. Fix with `TWeakObjectPtr` / `CreateWeakLambda` (the file already uses that pattern at `T66UITexturePoolSubsystem.cpp:147`).
- **HIGH** ‚Äî `Source\T66\Core\T66WebImageCache.cpp:79` ‚Äî `AsyncTask(ENamedThreads::GameThread, [this, Url, Data, bSuccess](){ ‚Ä¶ CachedTextures.Add ‚Ä¶ OnWebImageReady.Broadcast });`. Same issue on game-thread continuation.
- **HIGH** ‚Äî `Source\T66\UI\T66SlateTextureHelpers.cpp:94` ‚Äî `Pool->RequestTexture(..., [&Brush](UTexture2D* Loaded){ Brush.SetResourceObject(Loaded); })`. Captures `Brush` by reference; if any caller passes a stack-local brush the async write hits freed memory.
- **MEDIUM** ‚Äî `Source\T66\Core\T66UITexturePoolSubsystem.cpp:221` ‚Äî waiters with `bHasRequester == false` invoke their `TFunction` ungated.
- **LOW** ‚Äî `Source\T66Mini\Private\Core\T66MiniLeaderboardSubsystem.cpp:77` ‚Äî `Deinitialize() { delete SteamBridge; SteamBridge = nullptr; }` without explicit cancel/drain.

Backend HTTP path is mostly safe ‚Äî `T66BackendAccountApi.cpp:15` and friends use `BindUObject(this, ‚Ä¶)`.

### 3.2. Network JSON parsing

These crash (or `ensure`/`check` depending on engine build) if the backend returns a body without expected shape (outage page, partial response, schema bump):

- **HIGH** ‚Äî `Source\T66\Core\Backend\T66BackendLeaderboardApi.cpp:312-314, 485, 497-553` ‚Äî multiple `Json->GetNumberField` / `GetStringField` / `GetBoolField` for `rank`, `total_entries`, and per-entry fields with no `HasField` or `TryGet*` guard.
- **HIGH** ‚Äî `Source\T66\Core\Backend\T66BackendAccountApi.cpp:41` ‚Äî `Json->GetStringField(TEXT("restriction"))` with no guard.
- **HIGH** ‚Äî `Source\T66\Core\Backend\T66BackendRunApi.cpp:447` ‚Äî `Json->GetStringField(TEXT("status"))`.
- **HIGH** ‚Äî `Source\T66\Core\Backend\T66BackendRunSummaryParser.cpp:29-36` ‚Äî `hero_id`, `companion_id`, `display_name` read without the `HasField ? ...` guard that lines 18-27 in the same function use.

Fix uniformly: `TryGetStringField` / `TryGetNumberField` everywhere, with explicit defaults.

- **MEDIUM** ‚Äî `Source\T66\Core\T66SaveSubsystem.cpp:124-126` ‚Äî `Cast<UT66RunSaveGame>(Loaded)`. Returns `nullptr` for wrong class / corrupted slot. Crash is downstream-caller dependent.
